🎯 Úvod

Čo je tento projekt?

Toto je webová aplikácia na pomoc pri rozlúštení historických šifier - konkrétne typu zvaného nomenclátor.

Čo je nomenclátor?

Nomenclátor je starý šifrovací systém (používaný v 15.-19. storočí), ktorý kombinuje:

  1. Substitúciu znakov - každé písmeno sa nahradí číslom alebo symbolom
  2. Homofóny - jedno písmeno môže mať viacero rôznych šifier (napr. písmeno "e" môže byť "23", "45", alebo "67")
  3. Deception tokeny (nulls/klamače) - falošné čísla, ktoré nič neznamenajú, len zmätú kryptoanalytika

Prečo je to ťažké rozlúštiť?

Bez kľúča (tabuľky, ktorá hovorí, čo znamená ktoré číslo) je to veľmi náročné. Frekvenčná analýza je limitovaná kvôli homofónom a klamačom.

Čo aplikácia robí?

Pomáha kryptoanalytikovi postupne vytvoriť kľúč spárovaním:

  • OT (Original Text) = pôvodný/otvorený text (normálne písmená, napríklad "hello")
  • ZT (Ziffertext) = zašifrovaný text (čísla alebo symboly, napríklad "32 156 67 67 89")

Hlavné funkcie:

  1. Parsovanie ZT - rozdelí šifrovaný text na tokeny (čísla)
  2. Frekvenčná analýza - počíta, ako často sa vyskytujú písmená v OT a tokeny v ZT
  3. Automatické navrhovanie - navrhuje párovanie OT ↔ ZT podľa frekvencií
  4. Manuálne upravovanie - používateľ môže manuálne potvrdiť, zamietnuť alebo upraviť párovanie
  5. Multi-key režim - podporuje homofóny (jedno písmeno → viacero tokenov)
  6. Označovanie klamačov - tokeny, ktoré nič neznamenajú, sa dajú označiť zátvorkami
  7. Drag & Drop - možnosť presúvania tokenov, zlučovania buniek
  8. Persistencia - ukladá prácu do localStorage, aby sa dala pokračovať neskôr

📚 Základné Koncepty a Typy

Typy Dát (TypeScript)

Toto sú základné "kocky", z ktorých je postavená aplikácia.

OTChar (Original Text Character)

Reprezentuje jedno písmeno v otvorenom texte (normálny text).

type OTChar = {
  char: string;           // Písmeno (napr. "a", "b", "c")
  occurrenceNumber: number; // Poradové číslo výskytu tohto písmena (1, 2, 3...)
}

Príklad:

Text: "hello"
OTChars:
[
  { char: "h", occurrenceNumber: 1 },
  { char: "e", occurrenceNumber: 1 },
  { char: "l", occurrenceNumber: 1 },
  { char: "l", occurrenceNumber: 2 },  // druhé "l"
  { char: "o", occurrenceNumber: 1 }
]

Kedy sa používa: Všade, kde pracujeme s pôvodným textom. OT text sa rozdelí na polia OTChar objektov.

ZTToken (Ziffertext Token)

Reprezentuje jeden šifrovaný token (číslo alebo symbol v zašifrovanom texte).

type ZTToken = {
  id: string;   // Unikátny identifikátor (napr. "zt_0", "zt_1")
  text: string; // Obsah tokenu (napr. "32", "156", "abc")
}

Príklad:

ZT text: "32 156 67 67 89"
ZT Tokeny:
[
  { id: "zt_0", text: "32" },
  { id: "zt_1", text: "156" },
  { id: "zt_2", text: "67" },
  { id: "zt_3", text: "67" },
  { id: "zt_4", text: "89" }
]

Kedy sa používa: Po parsovaní ZT textu. Každý token predstavuje jednu "šifru".

KeysPerOTMode

Určuje, koľko rôznych šifier môže mať jedno písmeno.

type KeysPerOTMode = 'single' | 'multi';

  • 'single': jedno písmeno = jedna šifra (jednoduchá substitúcia)
    Príklad: "e" je vždy "23"

  • 'multi': jedno písmeno = viacero šifier (homofóny)
    Príklad: "e" môže byť "23", "45", alebo "67"

Kedy sa používa: Užívateľ si vyberie režim na začiatku. Ovplyvňuje všetky mappingy a výpočty.

LockedKeys

Uložené potvrdené párovanie medzi OT písmenom a ZT tokenmi.

type LockedKeys = Record<string, string | string[]>;

Príklad (single-key režim):

{
  "h": "32",
  "e": "156",
  "l": "67",
  "o": "89"
}

Príklad (multi-key režim):

{
  "e": ["23", "45", "67"],   // "e" môže byť 3 rôzne tokeny
  "t": ["12", "34"],
  "a": "56"                   // môže byť aj string, ak má len 1 token
}

Kedy sa používa: Keď užívateľ "zamkne" párovanie (potvrdí, že toto písmeno = tento token).

SelectionMap

Dočasné nepotvrdené kandidáty - návrhy od algoritmu alebo užívateľa.

type SelectionMap = Record<string, string | string[]>;

Štruktúra rovnaká ako LockedKeys, ale tieto párovanie ešte nie sú potvrdené.

Príklad:

{
  "h": "32",      // Navrhované párovanie
  "e": ["156", "67"]  // Multi-key návrh
}

Kedy sa používa: Pri automatickom navrhovaní alebo pri manuálnom výbere pred potvrdením.

Column

Reprezentuje bunku v mapping tabuľke - spája OT písmeno s jeho ZT tokenmi.

type Column = {
  otChar: OTChar | null;     // OT písmeno (alebo null = prázdna bunka)
  ztTokenIndices: number[];  // Indexy ZT tokenov priradených k tomuto OT
}

Príklad:

// Bunka pre písmeno "h", priradené k tokenu na indexe 0
{
  otChar: { char: "h", occurrenceNumber: 1 },
  ztTokenIndices: [0]  // token "32" je na indexe 0
}

// Multi-key príklad: "e" má 3 tokeny
{
  otChar: { char: "e", occurrenceNumber: 1 },
  ztTokenIndices: [5, 12, 18]  // "e" je na 3 pozíciách v ZT
}

Kedy sa používa: Hlavná dátová štruktúra pre mapping tabuľku. Pole Column[][] je celá tabuľka.

Candidate

Jeden kandidát pre párovanie - návrh, že toto OT písmeno by mohlo byť tento ZT token.

type Candidate = {
  text: string;      // Text ZT tokenu (napr. "32")
  score?: number;    // Voliteľné: skóre kvality párovania (vyššie = lepšie)
}

Príklad:

[
  { text: "32", score: 0.95 },   // Veľmi dobrý kandidát
  { text: "156", score: 0.75 },  // Stredný kandidát
  { text: "67", score: 0.45 }    // Slabý kandidát
]

Kedy sa používa: Pri automatickom navrhovaní - algoritmus vygeneruje zoznam kandidátov pre každé OT písmeno, zoradený podľa skóre.

ParseMode

Spôsob, ako sa parsuje ZT text (rozdelí na tokeny).

type ParseMode = 'separator' | 'fixed';

  • 'separator': tokeny sú oddelené oddeľovačom (napr. medzera, čiarka)
    Príklad: "32 156 67" → ["32", "156", "67"]

  • 'fixed': každý token má fixnú dĺžku (napr. 2 znaky)
    Príklad: "321567" (dĺžka=2) → ["32", "15", "67"]

Kedy sa používa: Užívateľ si vyberie na začiatku podľa formátu šifry.

BracketedIndices

Pole indexov ZT tokenov, ktoré sú označené ako klamače (deception tokens/nulls).

type BracketedIndices = number[];

Príklad:

[1, 3, 7]  // Tokeny na indexoch 1, 3, 7 sú klamače

Kedy sa používa: Klamače sa zobrazujú v hranatých zátvorkách [token] a ignorujú sa pri mappingu.

Základné konštanty

MAX_MULTI_KEYS

const MAX_MULTI_KEYS = 10;

Čo to je: Maximálny počet rôznych tokenov, ktoré môže mať jedno písmeno v multi-key režime.

Kedy sa používa: Validácia - ak písmeno má viac ako 10 tokenov, aplikácia zobrazí warning.

analysis.worker.ts

Účel: Web Worker pre frekvenčnú analýzu

Lokácia: src/workers/analysis.worker.ts

Worker: self.onmessage = (e) => { ... }

Vstup (cez postMessage):

{
  otGroups: OTChar[],
  ztTokens: ZTToken[],
  bracketedIndices: number[],
  lockedKeys: LockedKeys,
  keysPerOTMode: KeysPerOTMode
}

Výstup (cez postMessage späť):

{
  candidates: Record<string, Candidate[]>
}

Čo robí:

  1. Prijme dáta:

    • Main thread pošle postMessage(data)
    • Worker prijme cez onmessage

  2. Spustí analýzu:

    • Volá analyze() funkciu z analyzer.ts
    • Výpočet trvá 10-500ms (podľa veľkosti dát)

  3. Pošle výsledky:

    • Worker pošle postMessage({ candidates })
    • Main thread prijme v useAnalysis() hook

Prečo Web Worker?

  • Frekvenčná analýza je CPU-intensive
  • Bez workeru by UI zamrzol (blocking operation)
  • Worker beží v separátnom threade → UI zostáva responzívny

Príklad komunikácie:

// Main thread
worker.postMessage({
  otGroups: [...],
  ztTokens: [...],
  bracketedIndices: [],
  lockedKeys: {},
  keysPerOTMode: 'single'
});

// Worker thread
self.onmessage = (e) => {
  const result = analyze(e.data.otGroups, ...);
  self.postMessage({ candidates: result });
};

// Main thread
worker.onmessage = (e) => {
  setCandidates(e.data.candidates);  // Update UI
};

Kedy sa používa: Automaticky pri každej zmene OT/ZT/locks (debounced 300ms).

🚀 Výkonnostné Optimalizácie

1. Web Worker

  • Analýza beží v samostatnom vlákne
  • UI ostáva responzívne aj pri veľkých dátach

2. React.memo

  • Komponenty sa prerenderujú len ak sa zmenia ich props
  • MappingTable, OTCell, ZTToken sú memozované

3. Virtualizácia (react-window)

  • MappingTable vykreslí len viditeľné bunky
  • Pre 1000+ buniek toto šetrí pamäť a zrýchli rendering

4. Debouncing

  • Auto-refresh analýzy je debounced (čaká kým prestaneš písať)
  • Šetrí výpočtový čas

5. useMemo / useCallback

  • Výpočty ako columns, otRows, candidates sú cachovane
  • Preratajú sa len keď sa zmenia dependencies

6. Selective Re-renders

  • activeDragType, activeZtTokenIndex sa prechádza ako props
  • Vyhýba sa useDndContext() v každej bunke (tisíce re-renders)

🔍 Debugging Tips

Ako nájsť problém?

  1. Status Message: Pozri sa na status message pod ZT textarea
  2. Bracket Warning: Červený box upozorní na bracket problémy
  3. Selection Error: Upozornenie pri preview selection
  4. Red Cells: Bunky s chybami sú červené
  5. KeyTable Errors: Red text vedľa OT znaku vysvetľuje chybu

Časté Problémy

"Warning: too many ZT tokens"

Príčina: Máš viac ZT tokenov ako OT znakov Riešenie: Označ niektoré tokeny ako deception (bracket)

"Incomplete group: missing X characters"

Príčina: Fixed-length mode, ZT dĺžka nie je deliteľná groupSize Riešenie: Skontroluj ZT text (možno chýba znak)

"(multiple keys)" v KeyTable

Príčina: Single-key mode, ale jeden OT znak má viacero rôznych ZT tokenov Riešenie: Skontroluj mriežku, možno zlý lock alebo deception token

"(lock mismatch)" v KeyTable

Príčina: Locked token sa nezhoduje s aktuálnou alokáciou Riešenie: Unlock a re-lock, alebo oprať deception tokeny

"(duplicate)" v KeyTable

Príčina: Rovnaký ZT token je použitý pre rôzne OT znaky Riešenie: Jeden z nich musí byť chyba - skontroluj ktorý

🎓 Pre Začiatočníkov - ELI5 (Explain Like I'm 5)

Predstav si:

Máš tajnú správu (ZT), kde sú písmená nahradené číslami:

12 34 56 78 90

A máš čiastočne rozlúštený text (OT):

H E L L O

Otázka: Ktoré číslo znamená ktoré písmeno?

Aplikácia ti pomáha:

  1. Spočíta koľkokrát sa každé písmeno vyskytuje

  2. Spočíta koľkokrát sa každé číslo vyskytuje

  3. Porovná frekvencie

  4. Navrhne párovanie:

    • H sa vyskytuje 1x → 12 sa vyskytuje 1x → H = 12 (score 100%)
    • E sa vyskytuje 1x → 34 sa vyskytuje 1x → E = 34 (score 100%)
    • L sa vyskytuje 2x → 56 sa vyskytuje 1x → L = 56? (score 50%)
    • L sa vyskytuje 2x → 78 sa vyskytuje 1x → L = 78? (score 50%)

  5. Zobrazí mriežku kde vidíš párovanie

  6. Umožní ti opraviť chyby (drag & drop, lock správne páry)

  7. Výsledok: Kompletný kľúč H=12, E=34, L=56, L=78, O=90
